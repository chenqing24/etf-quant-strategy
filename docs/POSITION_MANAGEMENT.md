# ETF量化策略 - 仓位管理文档

## 1. 概述

本文档记录ETF量化策略的仓位管理规则，包括持仓数量、权重分配、调仓逻辑等。

---

## 2. 核心参数

| 参数 | 默认值 | 范围 | 说明 |
|------|:------:|:----:|------|
| `hold_count` | 2 | 1~5 | 同时持仓ETF数量 |
| `weights` | (0.5, 0.5) | - | 仓位权重分配 |
| `rebalance_days` | 5 | 3~30 | 调仓周期(天) |

---

## 3. 持仓规则

### 3.1 持仓数量

| 持仓数量 | 特点 | 适用场景 |
|:--------:|------|----------|
| 1只 | 集中, 高波动 | 极度自信 |
| **2只** | **平衡 (默认)** | **一般推荐** |
| 3-5只 | 分散, 低波动 | 风险厌恶 |

**当前配置**: 持仓Top2

### 3.2 仓位权重

#### 等权配置 (默认)

```python
weights = (0.5, 0.5)  # 各50%
```

| 持仓位 | 权重 | 资金占比 |
|:------:|:----:|:--------:|
| 第1只 | 50% | 0.5 |
| 第2只 | 50% | 0.5 |

#### 动态配置选项

```python
# 方案1: 龙头优先
weights = (0.6, 0.4)  # 第1只60%, 第2只40%

# 方案2: 适度分散
weights = (0.4, 0.3, 0.3)  # 3只各40%/30%/30%

# 方案3: 极度分散 (需hold_count=4)
weights = (0.3, 0.25, 0.25, 0.2)
```

#### 权重使用逻辑

```python
# 买入时应用权重
for i, (code, s, price) in enumerate(candidates[:config.hold_count]):
    w = config.weights[i] if i < len(config.weights) else 0.5
    shares = (total_equity * w) / price
```

**默认行为**: 如果持仓数 < weight长度，多余的权重不使用

## 4. 调仓规则

### 4.1 调仓周期

| 调仓周期 | 收益 | 回撤 | 夏普 | 特点 |
|:--------:|-----:|-----:|-----:|------|
| 3天 | +25.7% | -25.5% | 0.9 | 交易频繁, 成本高 |
| **5天** | **+51.9%** | **-22.6%** | **2.07** | **最佳平衡** |
| 10天 | +92.2% | -53.8% | 0.35 | 收益高, 回撤大 |
| 15天 | +41.8% | -54.2% | 0.25 | 交易少, 效果差 |

**当前配置**: 5天调仓

### 4.2 调仓触发条件

```python
# 每次调仓日检查
if market_ok and date_idx % rebalance_days == 0:
    # 1. 选出分数TopN的ETF
    # 2. 卖出不在TopN的持仓
    # 3. 买入新进入TopN的ETF
```

### 4.3 调仓流程

```
调仓日(T):
├── 1. 检查持仓
│   ├── 持仓ETF是否在新的Top2?
│   └── 是否有新的高分ETF可买入?
├── 2. 卖出 (不在Top2的)
│   ├── 卖出代码A
│   └── 卖出代码B
└── 3. 买入 (新进入Top2的)
    ├── 买入代码C (按权重分配)
    └── 买入代码D (按权重分配)
```

## 5. 风控规则

### 5.1 止损/止盈

| 参数 | 值 | 说明 |
|------|-----|------|
| `stop_loss` | -10% | 止损线 |
| `stop_gain` | +15% | 止盈线 |
| `max_hold_days` | 15天 | 最大持仓天数 |

**触发逻辑**:
```
if pnl <= -10% → 止损卖出
if pnl >= +15% → 止盈卖出
if hold_days >= 15天 → 超时卖出
```

### 5.2 市场过滤

| 参数 | 值 | 说明 |
|------|-----|------|
| `market_ma` | 60 | 市场均线周期 |
| `enable_market_filter` | True | 是否启用 |

**过滤逻辑**:
```
沪深300 > MA60 → 做多 (可持仓)
沪深300 <= MA60 → 空仓 (全部卖出)
```

### 5.3 交易成本

| 成本项 | 费率 |
|--------|------|
| 手续费 | 万3 (0.0003) |

**扣除方式**:
```python
# 卖出时扣除手续费
equity *= (1 + pnl) * (1 - fee_rate)
```

## 6. 完整配置示例

### 6.1 默认配置 (最保守)

```python
config = StrategyConfig(
    # 持仓
    hold_count=2,
    weights=(0.5, 0.5),
    
    # 调仓
    rebalance_days=5,
    
    # 风控
    stop_loss=-0.10,
    stop_gain=0.15,
    max_hold_days=15,
    
    # 市场
    market_ma=60,
    enable_market_filter=True,
    
    # 成本
    fee_rate=0.0003,
)
```

### 6.2 高收益配置

```python
config = StrategyConfig(
    hold_count=2,
    weights=(0.6, 0.4),  # 龙头优先
    rebalance_days=10,   # 减少调仓次数
    enable_market_filter=False,  # 不过滤 (但回撤大)
)
```

### 6.3 低回撤配置

```python
config = StrategyConfig(
    hold_count=3,         # 更多持仓
    weights=(0.4, 0.35, 0.25),  # 更分散
    rebalance_days=3,     # 更频繁调仓
    max_hold_days=10,     # 更快止损
)
```

## 8. 常见问题

**Q: 为什么只持仓2只?**
> 持仓太多会分散收益，2只是风险与收益的平衡点。回测显示更多持仓不会显著降低回撤。

**Q: 权重可以改吗?**
> 可以。默认(0.5, 0.5)是等权配置，也可以改成(0.6, 0.4)给予高分ETF更多仓位。

**Q: 调仓太频繁会不会佣金太多?**
> 会。5天调仓每次佣金约0.03%，每月约6次，全年约0.36%。这是合理的换手成本。

**Q: 止损止盈触发了怎么办?**
> 触发后立即按收盘价卖出，不等待调仓日。
