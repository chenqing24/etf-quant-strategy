## 一、问题背景

**核心问题**：推荐价格使用历史收盘价（1.101），而非实时价格（1.207）

**根因分析**：
1. `report_generator.py` 相对导入路径错误
2. 热数据管理器和交易校验器导入失败
3. 预热数据未传递给报告生成器

## 二、新增改进项

### 改进2.3: 智能推荐价格算法

**问题**：推荐价格非黑即白（纯信号价或纯实时价），不够灵活

**算法设计**：动态加权法

```
信号价 = 历史收盘价（策略评分时的价格）
实时价 = 热数据当前价格
偏差 = (实时价 - 信号价) / 信号价 × 100%

if 实时价不存在:
    推荐价 = 信号价
elif |偏差| < 3%:
    推荐价 = 实时价  # 市场稳定，跟随实际价格
elif 3% ≤ 偏差 < 8%:
    推荐价 = 信号价 × (1 + 偏差 × 0.3)  # 轻度调整
else:
    推荐价 = 信号价  # 偏离太大，以策略信号为准，发出警告
```

**效果示例**：

| 场景 | 信号价 | 实时价 | 偏差 | 推荐价 |
|------|--------|--------|------|--------|
| 稳定 | 1.101 | 1.115 | +1.3% | 1.115 |
| 轻度偏离 | 1.101 | 1.170 | +6.3% | 1.122 |
| 重度偏离 | 1.101 | 1.207 | +9.6% | 1.101 ⚠️ |

**测试用例**：

| ID | 名称 | 输入 | 预期输出 |
|----|------|------|----------|
| TC-09 | 稳定市场 | signal=1.101, live=1.115, deviation=+1.3% | 推荐价=1.115 |
| TC-10 | 轻度偏离 | signal=1.101, live=1.170, deviation=+6.3% | 推荐价≈1.122 |
| TC-11 | 重度偏离 | signal=1.101, live=1.207, deviation=+9.6% | 推荐价=1.101 + 警告 |
| TC-12 | 无实时价 | signal=1.101, live=None | 推荐价=1.101 |

**文件**：`src/analysis/report_generator.py`
