# ETF量化系统 - 架构改进说明（新手版）

> 用简单的比喻解释为什么需要改进

---

## 一、当前系统的问题

### 1.1 配置乱放 - 像杂乱的抽屉

**现状**: 策略配置写在代码里，改一个参数要翻代码

```python
# 当前: 配置写在代码里
config = BacktestConfig(
    threshold=0.8,       # 这个参数在哪？
    stop_loss=-0.05,    # 还有这个
    stop_profit=0.10,   # 还有...
)
```

**问题**: 
- 想试试"阈值0.85"？要改代码
- 想对比不同参数？每个都改一遍
- 记不清改了什么？很难回溯

**改进后**: 所有配置在一个文件里，像整理好的文件夹

```yaml
# 改进后: 配置在YAML文件
strategy:
  threshold: 0.8
  stop_loss: -0.05
  stop_profit: 0.10
```

---

### 1.2 风控散落 - 像没有保安的大楼

**现状**: 止损、止盈、仓位限制的代码分布在多个地方

```
executor.py  → 有止损代码
engine.py   → 有止盈代码  
scorer.py   → 有仓位检查
...         → 还有更多
```

**问题**:
- 改一个规则要改好几个地方
- 可能改漏了某个地方
- 测试不完整

**改进后**: 找一个"保安"（风控模块）统一管理

```
                    ┌─────────────────┐
                    │   风控模块      │
                    │  RiskManager   │
                    └────────┬────────┘
                             │
        ┌────────────────────┼────────────────────┐
        ▼                    ▼                    ▼
    ┌────────┐          ┌────────┐          ┌────────┐
    │ 止损   │          │ 止盈   │          │ 仓位   │
    └────────┘          └────────┘          └────────┘
```

---

### 1.3 策略紧耦合 - 像焊接死的零件

**现状**: 策略逻辑和回测引擎"焊"在一起

```
┌─────────────────────────────────┐
│     BacktestEngine (回测引擎)   │
│                                 │
│   里面"焊"着策略逻辑            │
│   - 怎么评分                    │
│   - 什么时候买                  │
│   - 什么时候卖                  │
└─────────────────────────────────┘
```

**问题**:
- 想换一种策略？要改整个引擎
- 想测试新策略？很麻烦
- 想同时跑多个策略？不可能

**改进后**: 策略变成"可拔插的插件"

```
┌─────────────────────────────┐
│     BacktestEngine (回测引擎)│
│                             │
│     有一个"插槽"给策略      │
└──────────────┬──────────────┘
               │
       ┌───────┴───────┐
       ▼               ▼
┌────────────┐   ┌────────────┐
│ 策略A     │   │ 策略B      │
│ Factor    │   │ Momentum   │
└────────────┘   └────────────┘
```

## 二、改进后的系统

### 2.1 配置管理 - 像乐高说明书

```yaml
# config/strategies/exp50.yaml
name: "Exp50策略"

# 评分因子
factors:
  - ADX       # 趋势强度
  - BB        # 布林带
  - SAR       # 抛物线
  - OBV       # 成交量

# 因子权重
weights:
  ADX: 0.5
  BB: 0.2
  SAR: 0.15
  OBV: 0.15

# 风控规则
risk:
  stop_loss: -0.05     # 亏5%就跑
  stop_profit: 0.055   # 赚5.5%就跑
  max_position: 1      # 最多1只

# 执行规则
execution:
  threshold: 0.8       # 评分>0.8才买
  hold_days: 3         # 最多持3天
```

**好处**:
- 想改参数？打开文件改一行
- 想对比？复制文件改几个数
- 想回溯？每个版本都留着

---

### 2.2 风控模块 - 像大楼保安

```
                    ┌─────────────────────┐
                    │     风险管理员       │
                    │   RiskManager       │
                    └──────────┬──────────┘
                               │
        ┌──────────────────────┼──────────────────────┐
        │                      │                      │
        ▼                      ▼                      ▼
   ┌─────────┐           ┌─────────┐           ┌─────────┐
   │ 止损检查 │           │ 止盈检查 │           │ 仓位检查 │
   │ 亏5%跑  │           │ 赚5%跑  │           │ 最多1只 │
   └─────────┘           └─────────┘           └─────────┘
```

**好处**:
- 所有规则在一个地方
- 改一处全生效
- 容易测试和审计

---

### 2.3 策略接口 - 像手机充电口

```
                    ┌─────────────────────┐
                    │   手机 (回测引擎)    │
                    │                     │
                    │   统一充电口         │
                    └──────────┬──────────┘
                               │
        ┌──────────────────────┼──────────────────────┐
        │                      │                      │
        ▼                      ▼                      ▼
   ┌─────────┐           ┌─────────┐           ┌─────────┐
   │ 策略A   │           │ 策略B   │           │ 策略C   │
   │ 因子评分│           │ 动量    │           │ 均值回归│
   │ 插入即用│           │ 插入即用│           │ 插入即用│
   └─────────┘           └─────────┘           └─────────┘
```

**好处**:
- 想换策略？拔掉旧的，插上新的
- 想同时试多个？插多个
- 想开发新策略？按接口开发就行

---

## 三、类比总结

| 问题 | 当前状态 | 改进后 | 类比 |
|------|----------|--------|------|
| 配置管理 | 代码里乱放 | YAML统一管理 | 杂乱的抽屉 → 整理好的文件夹 |
| 风控逻辑 | 散落多处 | RiskManager统一 | 没有保安 → 有保安的大楼 |
| 策略复用 | 和引擎焊接 | 可插拔接口 | 焊接的零件 → 乐高积木 |

---

## 四、改进后的好处

| 改进项 | 能做什么 |
|--------|----------|
| 配置外部化 | 轻松切换策略、对比参数、回溯历史 |
| 风控独立 | 集中管理、容易测试、一处修改全生效 |
| 策略接口化 | 支持多策略、便于测试、容易扩展 |
