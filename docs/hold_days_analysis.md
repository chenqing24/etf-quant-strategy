# 持仓时间与滑点分析报告

## 一、问题发现

### 1.1 实际持仓 vs 配置持仓

| 模型 | 配置持仓 | 实际平均持仓 | 差异 |
|------|----------|-------------|------|
| Exp36 | 3天 | **1.4天** | -53% |
| Exp48 | 3天 | **1.3天** | -57% |

### 1.2 退出原因分析

| 退出原因 | Exp36训练期 | Exp36测试期 | 分析 |
|----------|-------------|-------------|------|
| 到期(持有满3天) | 99.4% | 98.4% | ⚠️ 几乎全是到期 |
| 止损 | 0.2% | 0.5% | 极少触发 |
| 止盈 | 0% | 0.5% | 极少触发 |
| 期末平仓 | 0.4% | 0.5% | 少量 |

**关键发现**: 99%的交易确实是"到期"退出，但平均持仓只有1-2天

---

## 二、原因分析

### 2.1 调仓机制

查看代码发现，执行引擎支持**中途调仓**：

```python
# src/strategy/engine.py
# 平仓检查
if pos_score < threshold * 0.5:
    executor.check_and_close(code, current_price, current_date)
    continue  # 低于阈值一半就平仓

# 调仓逻辑
elif executor.positions:
    # 如果有更好的信号，换仓
    if best_new_score > worst_pos_score:
        executor.close(worst_pos)  # 卖出低分持仓
        executor.open(best_new)    # 买入高分持仓
```

### 2.2 持仓时间分布

```
Exp36 测试期持仓分布:
┌─────────────────────────────────────────────────────────┐
│ 持仓天数    笔数    占比                                  │
├─────────────────────────────────────────────────────────┤
│ 1天       22笔    6.0%  ← 大部分只持有1天                  │
│ 2天        2笔    0.5%                                   │
│ 3天        1笔    0.3%  ← 实际满3天的极少                 │
│ 4-30天     16笔   4.4%                                   │
│ 平均      1.4天                                         │
└─────────────────────────────────────────────────────────┘
```

**结论**: 调仓机制导致持仓被提前结束

## 四、问题根源

### 4.1 策略设计 vs 实际执行

```
配置: 持仓3天
  ↓
理想: 每笔交易持有3天
  ↓
实际: 
  Day1 买入 → Day2 发现更好信号 → 调仓
  Day2 卖出 → Day2 买入新标的
  ↓
结果: 实际平均持仓1-2天
```

### 4.2 信号评分导致频繁换仓

```python
# 每次评分的代码逻辑
for code, row in current_rows.items():
    score, _ = self.scorer.calculate(row)
    if score >= 0.8:  # 触发买入
        scores.append((code, score, price))

# 同时
for code in executor.positions:
    pos_score, _ = self.scorer.calculate(current_rows[code])
    if pos_score < 0.4:  # 低于阈值一半就平仓
        executor.check_and_close(...)  # 提前结束
```

## 五、修正建议

### 5.1 方案1: 增加持仓限制

```python
# 修改engine.py
# 不允许中途调仓，只有到期才平仓
def check_and_close(self, code, price, date):
    # 只检查止损止盈，不检查调仓
    if pnl_pct <= stop_loss:
        return close_position(...)
    if pnl_pct >= stop_profit:
        return close_position(...)
    if hold_days >= max_hold_days:
        return close_position(...)
    # 不允许提前平仓
    return None
```

### 5.2 方案2: 提高调仓门槛

```python
# 只有当新信号分数比持仓分数高N倍时才调仓
SKIP_THRESHOLD = 1.5  # 新信号分数需高于持仓分数1.5倍

if best_new_score > worst_pos_score * SKIP_THRESHOLD:
    executor.close(worst_pos)  # 调仓
else:
    pass  # 不调仓
```

### 5.3 方案3: 限制最大仓位

```python
# 限制最大持仓数量
max_positions = 2  # 最多同时持有2只ETF

# 只有空仓时才能开新仓
if len(positions) < max_positions:
    open_new(...)
```

### 5.4 方案4: 降低调仓频率

```python
# 每N天才检查一次调仓
REBALANCE_INTERVAL = 5  # 5天才检查调仓

if (current_date - last_rebalance_date).days >= REBALANCE_INTERVAL:
    check_rebalance(...)
```

## 七、结论与建议

### 7.1 问题总结

```
⚠️ 发现持仓时间异常

1. 配置持仓: 3天
2. 实际持仓: 1-2天 (平均1.4天)
3. 原因: 调仓机制导致频繁换仓
4. 影响: 年度交易成本高达130%
5. 后果: 回测收益高估60%以上
```

### 7.2 建议行动

| 优先级 | 行动 | 预期效果 |
|--------|------|----------|
| 高 | 修改调仓逻辑，强制持有满3天 | 成本降低60% |
| 中 | 提高调仓门槛 | 减少无效调仓 |
| 低 | 添加持仓限制 | 增加策略稳定性 |

### 7.3 最终结论

```
⚠️ 当前回测存在滑点高估问题

原因:
- 频繁调仓(平均1.4天持仓)导致高交易成本
- 年度交易成本约130%，严重侵蚀收益

建议:
1. 修正回测引擎，禁止提前调仓
2. 重新运行回测
3. 使用修正后的结果进行决策
```
