## 二、各阶段详细说明

### 阶段0：初始化（首次）

| 步骤 | 操作 | 文件/位置 |
|------|------|-----------|
| 0.1 | 创建数据目录 | `etf_data_live/` |
| 0.2 | 创建交易日志 | `etf_trades.json` |
| 0.3 | 创建配置 | `etf_config.json` |
| 0.4 | 设置定期任务 | cron: 每10天 |
| 0.5 | 初始化微信推送 | (可选) |

### 阶段1：数据采集

| 步骤 | 操作 | 数据源 |
|------|------|--------|
| 1.1 | 检查数据目录 | 本地 `etf_data_50/` |
| 1.2 | 获取最新日期 | 文件最新行 |
| 1.3 | 判断是否需要更新 | vs 当前日期 |
| 1.4 | 采集新数据 | 腾讯API / Tushare |
| 1.5 | 增量更新 | 只更新缺失日期 |

### 阶段2：数据分析

| 步骤 | 操作 | 输出 |
|------|------|------|
| 2.1 | 加载最新数据 | 54只ETF |
| 2.2 | 计算技术指标 | MA/RSI/MACD |
| 2.3 | 7因子打分 | 分数+原因 |
| 2.4 | 筛选Top N | ≥6分ETF |
| 2.5 | 市场环境判断 | 上涨/震荡/下跌 |

### 阶段3：策略验证

| 步骤 | 操作 | 参数 |
|------|------|------|
| 3.1 | 多时段回测 | 2023-2025, 2024-2026 |
| 3.2 | 计算关键指标 | 收益/回撤/夏普 |
| 3.3 | 策略评分 | 综合评分 |
| 3.4 | 参数调优 | (可选) |

### 阶段4：生成报告

| 步骤 | 输出 | 格式 |
|------|------|------|
| 4.1 | 定量数据 | 表格 |
| 4.2 | 定性分析 | LLM论述 |
| 4.3 | 配置建议 | 买入/持仓/卖出 |
| 4.4 | 输出 | `etf_report_YYYYMMDD.txt` |

### 阶段5：交易执行

| 步骤 | 操作 | 说明 |
|------|------|------|
| 5.1 | 生成交易信号 | 买入/卖出/持有 |
| 5.2 | 计算仓位 | 本金/价格/数量 |
| 5.3 | 发送提醒 | 钉钉/微信(可选) |
| 5.4 | 等待用户确认 | 实际是否成交 |

### 阶段6：定期评估

| 触发条件 | 操作 |
|----------|------|
| 每10天 | 重新评分，决定是否调仓 |
| 止损触发 | 立即平仓提醒 |
| 止盈触发 | 移动止盈提醒 |
| 用户请求 | 手动触发 |

### 阶段7：收集结果

| 步骤 | 操作 |
|------|------|
| 7.1 | 询问用户实际成交价 |
| 7.2 | 询问当前持仓盈亏 |
| 7.3 | 记录交易日期/价格 |
| 7.4 | 计算vs预期偏差 |

### 阶段8：结果对比

| 对比项 | 数据来源 |
|--------|----------|
| vs 报告预期 | 报告中的情景分析 |
| vs 历史表现 | 上次/历史交易 |
| vs 基准 | 沪深300 |
| 偏差分析 | 超额/落后原因 |

### 阶段9：存储记录

| 文件 | 内容 |
|------|------|
| `etf_trades.json` | 所有交易记录 |
| `etf_reports/` | 历史报告副本 |
| `etf_performance.json` | 累计表现 |
| `etf_log.txt` | 操作日志 |

## 四、交互流程

### 首次使用
```
用户: "开始ETF量化投资"
AI: 
1. 检查数据目录
2. 初始化配置
3. 生成首份报告
4. 询问: "是否按建议执行?"
```

### 定期评估
```
用户: "10天到了，评估"
AI:
1. 加载最新数据
2. 重新打分
3. 判断是否调仓
4. 生成报告
5. 询问结果输入

```

```
3. 持有天数: ?
4. 实际盈亏: ?"

用户: [输入]
AI: 记录并分析vs预期
```

## 五、定期任务配置

### Cron设置
```bash
# 每10天下午3点评估
0 15 */10 * * cd /path/to/etf_strategy && python -c "from src.quant decision import run评估"

# 每周一检查持仓
0 9 * * 1 ...
```

### 提醒机制
- 评估前1天: 提醒准备
- 评估当天: 自动运行
- 交易信号: 即时通知
