- **12-Factor原则**：配置外部化，优于硬编码

## 三、具体怎么改？（改进方案）

### P0-1: 配置外部化

```
                    ┌─────────────────────────────────┐
                    │      目标: 配置从代码→YAML      │
                    └───────────────┬─────────────────┘
                                    │
    ┌───────────────────────────────┼───────────────────────────────┐
    │                               │                               │
    ▼                               ▼                               ▼
┌─────────┐                   ┌─────────┐                   ┌─────────┐
│ 旧方式  │                   │ 新方式  │                   │ 好处    │
│ config.py│                   │ YAML文件 │                   │         │
│ 中硬编码│                   │ config/  │                   │ 可切换  │
└─────────┘                   │ strategies/│               │ 可对比  │
                              │ exp50.yaml│               │ 可追溯  │
                              └─────────┘               └─────────┘
```

**小结**:
- 创建`config/strategies/exp50.yaml`等配置文件
- `ConfigLoader`类统一加载，支持缓存
- **关键代码**: `StrategyConfig` dataclass统一配置结构

### P0-2: 风控模块独立

```
                    ┌─────────────────────────────────┐
                    │   目标: 找一个"保安"统一管理   │
                    └───────────────┬─────────────────┘
                                    │
    ┌───────────────────────────────┼───────────────────────────────┐
    │                               │                               │
    ▼                               ▼                               ▼
┌─────────┐                   ┌─────────┐                   ┌─────────┐
│ 旧方式  │                   │ 新方式  │                   │ 好处    │
│ 分散多处│                   │ RiskManager│               │ 一处修改│
│ executor│                   │ 集中风控 │                   │ 全生效  │
│ engine  │                   │ ┌───────┐│                   │ 容易测试│
│ scorer  │                   │ │止损   ││                   │ 便于审计│
```

**小结**:
- `RiskManager`统一管理：止损、止盈、仓位、亏损限制
- 每个bar调用`check_exit()`，触发平仓
- **关键代码**: `RiskManager.check_exit(position, price)`返回`ExitSignal`

## 四、模块怎么通信？（接口契约）

```
                    ┌─────────────────────────────────┐
                    │         接口设计原则            │
                    └───────────────┬─────────────────┘
                                    │
    ┌───────────────────────────────┼───────────────────────────────┐
    │                               │                               │
    ▼                               ▼                               ▼
┌─────────┐                   ┌─────────┐                   ┌─────────┐
│ 输入    │                   │ 处理    │                   │ 输出    │
│ 类型定义 │                   │ 方法    │                   │ 错误码  │
│ dataclass│                  │ 职责明确│                   │ E1001   │
└─────────┘                   └─────────┘                   │ E2001   │
                                                            └─────────┘
```

**小结**:
- **类型安全**: 用`@dataclass`定义所有数据结构
- **错误码统一**: E1001(配置)、E2001(风控)、E2002(止损止盈)
- **接口示例**:
  ```python
  # 配置加载
  config = ConfigLoader.load("config/strategies/exp50.yaml")
  
  # 风控检查
  result = risk.check_entry(portfolio)  # 返回CheckResult
  signal = risk.check_exit(position, price)  # 返回ExitSignal或None
  ```

## 六、出问题怎么办？（风险控制）

```
                    ┌─────────────────────────────────┐
                    │         三层保障                │
                    └───────────────┬─────────────────┘
                                    │
    ┌───────────────────────────────┼───────────────────────────────┐
    │                               │                               │
    ▼                               ▼                               ▼
┌─────────┐                   ┌─────────┐                   ┌─────────┐
│ 风险识别 │                   │ 回滚方案 │                   │ 熔断机制 │
│ 高/中/低│                   │ Git/配置│                   │ 防止级联│
│ 恢复步骤│                   │ 特性开关│                   │ 失败3次  │
└─────────┘                   └─────────┘                   │ 打开熔断 │
                                                            └─────────┘
```

**小结**:
- **风险分级**: R1(改动范围大)🔴, R2(配置不兼容)🟡, R3(时间紧迫)🟡
- **回滚方案**: 
  - 方案A: `git revert` 回滚代码
  - 方案B: 切换到 `default.yaml`
  - 方案C: `USE_NEW_RISK=true/false` 特性开关
- **熔断保护**: API连续失败3次，进入熔断，30秒后尝试恢复
