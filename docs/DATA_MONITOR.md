# 数据质量监控设计

**日期**: 2026-05-27
**状态**: 设计完成

---

## 背景

数据质量是量化策略的基石，需要建立监控机制确保数据长期可靠。

---

## 监控目标

1. **数据完整性**：确保66只ETF数据不缺失
2. **数据时效性**：确保数据每日更新
3. **数据正确性**：确保字段逻辑约束满足
4. **异常检测**：及时发现数据异常

---

## 监控指标

### 1. 数据完整性指标

| 指标 | 描述 | 告警阈值 |
|------|------|----------|
| ETF覆盖率 | 数据库ETF数量/目标ETF数量 | <95% |
| 数据行数 | 每只ETF数据行数 | <100行 |
| 时间覆盖 | 数据时间范围 | 缺失超过30天 |

### 2. 数据时效性指标

| 指标 | 描述 | 告警阈值 |
|------|------|----------|
| 数据新鲜度 | 最新数据日期距今天数 | >2天 |
| 更新成功率 | 每日更新成功比例 | <90% |

### 3. 数据正确性指标

| 指标 | 描述 | 告警阈值 |
|------|------|----------|
| 字段约束 | high>=close>=low | 任何违规 |
| 价格合理性 | close>0 且 <1000 | 任何异常 |
| 数值连续性 | 日涨幅/跌幅 | >20% |

---

## 监控方案

### 1. 每日定时检查（Cron）

```bash
# 每日收盘后（15:30）执行
30 15 * * 1-5 cd /path/to/etf_strategy && python -m pytest tests/test_data_field_order.py -q
```

### 2. 每周抽样验证

```bash
# 每周一抽取3只ETF进行交叉验证
python scripts/cross_validate_data.py --sample 3 --verbose
```

### 3. 每月全面检查

```bash
# 每月1日进行全面检查
python -m pytest tests/ -q --tb=short
```

## 告警机制

### 告警级别

| 级别 | 说明 | 触发条件 |
|------|------|----------|
| 🔴 严重 | 数据严重异常 | ETF数量<60 或 数据过期>7天 |
| 🟡 警告 | 数据轻度异常 | ETF数量<65 或 数据过期>2天 |
| 🟢 提示 | 数据需要关注 | 抽样发现异常 |

### 告警方式

1. **钉钉通知**：通过 `qwenpaw channels send` 发送告警
2. **日志记录**：写入 `etf_reports/cache/` 目录
3. **状态文件**：生成 `data_quality_status.json`
