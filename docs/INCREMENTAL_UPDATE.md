# 增量更新机制设计

**日期**: 2026-05-27
**状态**: 设计中

---

## 背景

当前数据获取方式为一次性批量采集，需要设计增量更新机制实现每日自动更新。

---

## 设计目标

1. **每日自动更新**：收盘后自动获取最新数据
2. **增量更新**：只获取新数据，不重复获取历史数据
3. **容错机制**：网络失败时自动重试
4. **限流保护**：避免频繁请求被封禁

---

## 架构设计

```
每日定时触发
    ↓
DataSourceRouter.fetch_incremental()
    ↓
遍历所有ETF
    ↓
获取本地最新日期
    ↓
从腾讯API获取新数据
    ↓
写入SQLite（append模式）
    ↓
验证数据完整性
```

---

## 核心接口

### DataSourceRouter增量更新

```python
class DataSourceRouter:
    def fetch_incremental(
        self,
        codes: List[str] = None,
        days: int = 7
    ) -> Dict[str, pd.DataFrame]:
        """
        增量获取数据
        
        Args:
            codes: ETF代码列表，None表示全部ETF
            days: 增量获取的天数，默认7天
        
        Returns:
            Dict[str, pd.DataFrame]: 各ETF的新增数据
        """
```

### 增量更新流程

1. 获取本地数据库中每只ETF的最新日期
2. 计算需要获取的起始日期 = 最新日期 + 1天
3. 调用腾讯API获取数据
4. 过滤出新增数据（日期 > 本地最新日期）
5. 追加写入SQLite

## 数据验证

### 验证规则

1. **日期连续性**：新数据日期必须 >= 本地最新日期
2. **字段完整性**：high/low/close 不能为空
3. **逻辑约束**：high >= close >= low
4. **价格合理性**：close > 0

### 异常处理

| 异常类型 | 处理方式 |
|----------|----------|
| 网络超时 | 重试3次，每次间隔10秒 |
| API返回空 | 记录日志，跳过该ETF |
| 数据异常 | 记录异常，继续下一只 |
