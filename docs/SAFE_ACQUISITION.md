# ETF数据采集安全规范 v1.0

> 确保数据采集稳定可靠，避免API限流或封号

## 1. 采集策略

### 1.1 间隔控制

| 池类型 | ETF数量 | 间隔范围 | 说明 |
|--------|---------|----------|------|
| **核心池** | ~50只 | 1.0~2.0秒 | 规模大、成交高 |
| **扩展池** | ~50只 | 1.5~3.0秒 | 规模中等 |
| **跨境池** | ~30只 | 3.0~6.0秒 | 海外ETF（不在本系统范围） |

### 1.2 分批处理

| 批次 | 数量 | 批间休息 |
|------|------|----------|
| 核心池 | 20只/批 | 60秒 |
| 扩展池 | 15只/批 | 60秒 |
| 全量 | 全部 | 每池间休息60秒 |

### 1.3 采集时间

| 任务 | 时间 | 说明 |
|------|------|------|
| 核心池 | 凌晨3:00 | 低峰期 |
| 扩展池 | 凌晨4:00 | 低峰期 |
| 全量验证 | 每周日凌晨3:00 | 全部采集 |

## 2. 多渠道验证

### 2.1 渠道配置

| 渠道 | API地址 | 用途 | 超时 |
|------|---------|------|------|
| **腾讯** | `qt.gtimg.cn` | 主数据源 | 5秒 |
| **新浪** | `hq.sinajs.cn` | 交叉验证 | 5秒 |
| **东方财富** | `push2.eastmoney.com` | 备用验证 | 10秒 |

### 2.2 验证规则

```
腾讯名称 == 新浪名称 → 验证通过 ✅
腾讯名称 != 新浪名称 → 验证不一致 ⚠️（记录，不阻断）
两个都失败 → 标记失败，加入重试队列
```

### 2.3 响应格式

**腾讯API**：
```
v_sh510300="51~沪深300ETF华泰柏瑞~510300~3.856~..."
              ↑ 字段1是名称
```

**新浪API**：
```
var hq_str_sh510300="沪深300ETF华泰柏瑞,3.856,3.807,...";
                    ↑ 引号内逗号前是名称
```

## 3. 失败处理

### 3.1 重试机制

| 重试次数 | 延迟时间 | 说明 |
|---------|----------|------|
| 第1次 | 60秒 | 1分钟 |
| 第2次 | 300秒 | 5分钟 |
| 第3次 | 1800秒 | 30分钟 |

### 3.2 持久化重试队列

```sql
-- 重试队列表
CREATE TABLE etf_name_retry_queue (
    code TEXT PRIMARY KEY,
    attempt_count INTEGER DEFAULT 0,
    last_error TEXT,
    status TEXT DEFAULT 'pending',
    priority INTEGER DEFAULT 0,
    next_retry_at TEXT
);
```

### 3.3 失败处理流程

```
采集失败
    ↓
加入重试队列（持久化到数据库）
    ↓
指数退避等待
    ↓
重试最多3次
    ↓
全部失败 → 标记为 failed
    ↓
连续3天失败 → 钉钉告警
```

## 4. 监控告警

### 4.1 监控指标

| 指标 | 说明 | 计算 |
|------|------|------|
| 成功率 | 成功/总数 | 24小时窗口 |
| 平均耗时 | 采集耗时均值 | 毫秒 |
| 失败数量 | 失败的ETF数量 | - |

### 4.2 告警阈值

| 条件 | 级别 | 动作 |
|------|------|------|
| 成功率 < 90% | ⚠️ 警告 | 日志记录 |
| 失败数量 > 10 | 🔴 严重 | 钉钉告警 |
| 同一只连续失败3天 | 🔴 严重 | 钉钉告警 |
| 平均耗时 > 5秒 | ⚠️ 警告 | 日志记录 |

## 5. API格式验证

### 5.1 格式检测

| 渠道 | 期望格式 | 检测 |
|------|----------|------|
| 腾讯 | `~名称~代码~价格` | 正则 `~[^~]+~[0-9]` |
| 新浪 | `"名称,价格"` | 正则 `"[^"]+",` |

### 5.2 格式变化处理

```
连续3次响应异常 → 判定为格式变化 → 发送告警
```

## 6. ETF生命周期

### 6.1 状态

| 状态 | 说明 |
|------|------|
| **active** | 正常采集 |
| **inactive** | 已停牌/退市 |
| **failed** | 连续失败 |

### 6.2 自动处理

| 场景 | 处理 |
|------|------|
| 新ETF发现 | 加入池 → 触发采集 |
| 名称变更 | 记录历史 → 告警 |
| 退市ETF | 标记 inactive |
| 连续失败3天 | 标记 failed → 告警 |

---

## 7. 安全边界

| 指标 | 安全阈值 | 说明 |
|------|----------|------|
| 单次请求间隔 | > 0.5秒 | 避免触发限流 |
| 单渠道请求频率 | < 60次/分钟 | 腾讯API限制 |
| 批次间休息 | > 30秒 | 给API恢复时间 |
| 单日总请求量 | < 200次 | 留有余量 |
