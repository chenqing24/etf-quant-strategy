## 二、更新规则

### 规则1：工具更新同步文档
**触发时机**：添加/修改脚本、Python函数、CLI命令

**必须更新**：
1. `docs/TOOLS.md` - 添加工具说明（名称、位置、用法）
2. `docs/INDEX.md` - 更新场景索引
3. `docs/MODULES.md` - 更新模块依赖（如果涉及）
4. `MEMORY.md` - 更新关键索引表

**示例**：
```bash
# 新增一个脚本 scripts/new_tool.py 后
# 1. 在 TOOLS.md 添加工具说明
# 2. 在 INDEX.md 更新场景
# 3. 在 MODULES.md 更新模块说明（如需）
# 4. 在 MEMORY.md 更新索引表
```

---

### 规则2：每次会话读取文档索引
**触发时机**：每次会话开始

**必须读取**：
1. `docs/TOOLS.md` - 避免重复造轮子
2. `docs/INDEX.md` - 快速定位功能
3. `docs/MODULES.md` - 理解代码结构

**目的**：确保使用最新工具信息，不依赖旧记忆

## 三、文档责任人

| 文档 | 责任人 | 说明 |
|------|--------|------|
| TOOLS.md | 每次添加工具的人 | 谁添加工具谁更新 |
| INDEX.md | 功能负责人 | 谁实现功能谁更新 |
| MODULES.md | 模块负责人 | 谁改模块谁更新 |

---

## 四、验证检查清单

每次提交前检查：
```
[ ] 是否添加工具/修改工具？
    → 是 → 更新 TOOLS.md
[ ] 是否有新场景/功能？
    → 是 → 更新 INDEX.md
[ ] 是否修改了模块？
    → 是 → 更新 MODULES.md
[ ] 是否修改了任何文档？
    → 是 → 更新 MEMORY.md 索引表
```
